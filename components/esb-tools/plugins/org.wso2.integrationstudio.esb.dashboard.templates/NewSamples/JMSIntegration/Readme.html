JMS Integration

About

This sample demonstrates some of the capabilities in WSO2 EI for integrating with JMS brokers.

Two modules get created in this sample. ESB Configs module contains a REST API called ‘OrderPaymentServiceAPI’, a JMS Inbound Endpoint called ‘OrderPaymentQueueInboundEP’, an Endpoint called ‘OrderPaymentQueueEP’ and two Sequences OrderPaymentQueueProcessSeq and OrderPaymentQueueErrorSeq. The Composite Exporter module is used to package and export the artifacts created within the ESB Config module.

The REST API receives HTTP request with XML payload and then it publishes to the OrderPaymentQueue queue. OUT_ONLY property in the mediation flow denotes the one way asynchronous invocation. Also, the client will receive a ‘202 Accepted’ response as the FORCE_SC_ACCEPTED property is set in the mediation flow.

JMS inbound endpoint periodically polls XML messages in the OrderPaymentQueue. It consumes available messages in the queue and injects them to the sequence. The sequence logs the message. The endpoint represents the JMS destination.

  1. Start ActiveMQ Broker

    In this example, we are using ActiveMQ Broker.

  2. Configure MI server

    Select the Embedded Micro Integrator Server Configuration Wizard from the tool bar and add the following config to the Edit embedded deployment.toml file section to enable the ActiveMQ sender and listener transport.

                                 [[transport.jms.sender]]
                                 name = "default"
                                 parameter.initial_naming_factory = "org.apache.activemq.jndi.ActiveMQInitialContextFactory"
                                 parameter.provider_url = "tcp://localhost:61616"
                                 parameter.connection_factory_name = "QueueConnectionFactory"
                                 parameter.connection_factory_type = "queue"
                                 parameter.cache_level = "producer"
                                 
                                 [[transport.jms.listener]]
                                 name = "myQueueConnectionFactory"
                                 parameter.initial_naming_factory = "org.apache.activemq.jndi.ActiveMQInitialContextFactory"
                                 parameter.provider_url = "tcp://localhost:61616"
                                 parameter.connection_factory_name = "QueueConnectionFactory"
                                 parameter.connection_factory_type = "queue"

    Add the following libraries (available in ACTIVEMQ_HOME/lib) to the Embedded Micro Integrator Server Configuration Wizard's Select libraries section by clicking the icon.

    If you are using ActiveMQ 5.8.0 or above

    activemq-broker-5.8.0.jar

    activemq-client-5.8.0.jar

    activemq-kahadb-store-5.8.0.jar

    geronimo-jms_1.1_spec-1.1.1.jar

    geronimo-j2ee-management_1.1_spec-1.0.1.jar

    geronimo-jta_1.0.1B_spec-1.0.1.jar

    hawtbuf-1.9.jar

    Slf4j-api-1.6.6.jar

    activeio-core-3.1.4.jar (available in the ACTIVEMQ_HOME/lib/optional directory)

    For Earlier versions of ActiveMQ

    activemq-core-5.5.1.jar

    geronimo-j2ee-management_1.0_spec-1.0.jar

    geronimo-jms_1.1_spec-1.1.1.jar

  3. Export project atrifacts & run

    Right click on the respective Composite Exporter module and select Export Project Atrifacts and Run from menu.

  4. Invoke the service

    Open the Postman application. If you do not have the application, download it from here : Postman

    Enter the request information as given below and click the Send.

    Method:

    POST

    URL:

    	                        	http://localhost:8290/orderpayment

    Payload:

    
                        {
                            "Payment": {
                                "Id": "1",
                                "Amount": "100"
                            }
                        }

    Headers:

                            		Content-Type: application/json

  1. Configure EI with Message Broker

    In this example we are using ActiveMQ broker. Refer to this and configure the EI to work with ActiveMQ Broker.

  2. Configure EI server

    If you haven't already attached the WSO2 EI runtime to WSO2 Integration Studio, follow the instructions given below.

    - Open the Integration Studio Getting Started and click Add Server.

    - In the ‘Define a New Server’ dialog box, expand the WSO2 folder, and select the respective product and version.

    - Click Next, and then click Browse (next to CARBON_HOME) and select the WSO2 EI distribution directory.

    - Click Next and review the default ports for the ESB profile of WSO2 EI. Typically, you can leave these unchanged, but if you are already running another server on these ports, specify unused ports here.

    - Click Finish. The ESB server of WSO2 EI is now connected to WSO2 EI Tooling.

  3. Deploy the ESB artifacts in the server

    - In the Servers tab, right-click the server and click Add and Remove.

    - Select the relevant project from the list, and click Add.

    - Click Finish. The sample is now deployed in the server.

  4. Start the server

    - Click the Start the server button on the toolbar.

    - If you are prompted to save changes to any of the artifact files, click Yes.

    - As the server starts, the Console tab will appear. You should see messages indicating that the C-App was successfully deployed.

  5. Invoke the service

    Use Postman as mentioned in above or following cURL command to invoke the service.

    curl -X POST -d @request.xml http://localhost:8280/orderpayment -H "Content-Type: application/json"

    The request.xml can be a file which has following content.

    
                        {
                            "Payment": {
                                "Id": "1",
                                "Amount": "100"
                            }
                        }
